# Guia de Upgrade — GEMS SDK 1.0.x → 1.1.0

Este guia lista, passo a passo, tudo o que você precisa fazer nos seus projetos
consumidores para subir de `@gabriel-mdias/angular-gems-sdk@1.0.x` para `1.1.0`.

> **Resumo:** a 1.1.0 é uma consolidação para produção. Há **breaking changes**
> em **autenticação (Keycloak)** e em **nomes de alguns inputs** (com aliases de
> compatibilidade). O restante é retrocompatível.

---

## 0. Atualizar o pacote

```bash
npm install @gabriel-mdias/angular-gems-sdk@1.1.0 --legacy-peer-deps
```

O `sweetalert2` agora é **dependency** da SDK (instala automaticamente). O
**Font Awesome** passou a ser declarado como **peerDependency opcional** — se
ainda não o tem, instale e registre o CSS (passo 3).

---

## 1. Estilos globais (nenhuma mudança se você já usa `styles.css`)

Continue com o import único:

```css
/* src/styles.css */
@import '@gabriel-mdias/angular-gems-sdk/styles.css';
```

Se você ainda importava caminhos antigos (`.../core/tokens/...`), troque pelo
import único acima.

---

## 2. HTTP / URL da API — use `provideGemsHttp` (recomendado)

**Antes (1.0.x):**

```ts
providers: [{ provide: GEMS_API_URL, useValue: 'https://api.seuprojeto.com' }];
```

**Depois (1.1.0):** continua funcionando, mas prefira o helper:

```ts
import { provideGemsHttp } from '@gabriel-mdias/angular-gems-sdk';

providers: [provideGemsHttp('https://api.seuprojeto.com')];
```

> ⚠️ **Novo comportamento:** se `GEMS_API_URL` **não** for provido e você usar
> um `GemsBaseStore` (ou `<gems-file-upload>`), o erro agora é **explícito e
> descritivo** (antes era um `NullInjectorError` genérico). Garanta o provider.

---

## 3. Ícones (Font Awesome) — declare explicitamente

Se ainda não tinha, os ícones (`fa-*`) dos componentes exigem o Font Awesome:

```bash
npm i @fortawesome/fontawesome-free
```

```jsonc
// angular.json → projects.<app>.architect.build.options.styles
"styles": [
  "src/styles.css",
  "node_modules/@fortawesome/fontawesome-free/css/all.min.css"
]
```

## 4. ⚠️ BREAKING — Autenticação (Keycloak) migrou para `/auth`

### 4.1. Imports de guard/diretiva mudaram de caminho

**Antes (1.0.x):**

```ts
import { gemsRoleGuard, GemsHasRoleDirective } from '@gabriel-mdias/angular-gems-sdk';
```

**Depois (1.1.0):**

```ts
import {
  gemsRoleGuard,
  GemsHasRoleDirective,
  provideGemsKeycloak,
} from '@gabriel-mdias/angular-gems-sdk/auth';
```

> Motivo: o `keycloak-angular` deixou de ser empacotado na raiz do pacote —
> agora só é carregado por quem importa de `/auth`. Projetos que **não** usam
> auth não carregam mais o Keycloak no bundle.

### 4.2. ⚠️ O token JWT NÃO é mais anexado automaticamente pelo `GemsBaseStore`

Antes, o `GemsBaseStore` injetava o `KeycloakService` e anexava o
`Authorization: Bearer` sozinho. **Isso foi removido.** Agora o token é anexado
por um **interceptor** registrado pelo `provideGemsKeycloak`.

**Adicione o setup do Keycloak no `app.config.ts`:**

```ts
import { provideHttpClient, withInterceptorsFromDi } from '@angular/common/http';
import { provideGemsKeycloak } from '@gabriel-mdias/angular-gems-sdk/auth';

export const appConfig: ApplicationConfig = {
  providers: [
    // ...
    provideHttpClient(withInterceptorsFromDi()), // necessário p/ o bearer interceptor
    provideGemsKeycloak({
      config: {
        url: 'https://auth.seuapp.com',
        realm: 'seu-realm',
        clientId: 'seu-client',
      },
      initOptions: {
        onLoad: 'check-sso',
        silentCheckSsoRedirectUri: window.location.origin + '/silent-check-sso.html',
      },
    }),
  ],
};
```

Crie também `public/silent-check-sso.html` (arquivo padrão do keycloak-js):

```html
<html>
  <body>
    <script>
      parent.postMessage(location.href, location.origin);
    </script>
  </body>
</html>
```

> **Se você já inicializava o Keycloak manualmente** (com `APP_INITIALIZER` +
> `KeycloakService.init`) e registrava o `KeycloakBearerInterceptor` por conta
> própria, pode manter o seu setup — nesse caso NÃO precisa do
> `provideGemsKeycloak`. O importante é que **algum** bearer interceptor esteja
> ativo, já que o `GemsBaseStore` não anexa mais o token.

### 4.3. Guard: semântica padrão e modo de papéis

- O guard e a diretiva agora usam **"qualquer papel"** (`any`) por padrão, de
  forma **consistente** (antes o guard exigia TODOS os papéis).
- Para exigir todos, use `mode: 'all'`:

```ts
// rota
{ path: 'admin', canActivate: [gemsRoleGuard],
  data: { roles: ['ADMIN', 'FINANCE'], roleMode: 'all' } }
```

```html
<!-- template -->
<div *gemsHasRole="['A','B']; mode: 'all'">Só com A e B</div>
```

> Se você dependia do comportamento antigo do guard (exigir TODOS), adicione
> `roleMode: 'all'` nas rotas afetadas.

## 5. ⚠️ BREAKING (com alias) — Nomes de inputs em inglês

Os valores/labels em português foram renomeados para inglês. Os **nomes antigos
continuam funcionando** marcados como `@deprecated` — atualize quando puder.

| Componente        | Antes (1.0.x)         | Agora (1.1.0)           |
| ----------------- | --------------------- | ----------------------- |
| `gems-input-date` | `formato="diaMesAno"` | `format="dayMonthYear"` |
| `gems-input-date` | `formato="fullData"`  | `format="fullDate"`     |
| `gems-input-date` | `formato="mesAno"`    | `format="monthYear"`    |
| `gems-input-date` | `formato="ano"`       | `format="year"`         |
| `gems-input-mask` | `maskType="telefone"` | `maskType="phone"`      |

```html
<!-- Antes -->
<gems-input-date formControlName="data" formato="diaMesAno" />
<gems-input-mask formControlName="tel" maskType="telefone" />

<!-- Depois -->
<gems-input-date formControlName="data" format="dayMonthYear" />
<gems-input-mask formControlName="tel" maskType="phone" />
```

> Os valores em português ainda são aceitos nesta versão, mas emitem aviso de
> depreciação e serão removidos numa versão futura.

## 6. Correções que você ganha de graça (sem ação necessária)

- **Formulários com `disabled`**: `control.disable()` agora desabilita de fato
  os campos `gems-input-text`, `gems-select` e `gems-textarea` (antes não).
- **`gems-select` com valores objeto**: agora seleciona corretamente; se usar
  objetos, informe um comparador: `[compareWith]="(a,b) => a.id === b.id"`.
- **`gems-modal`**: recebe foco ao abrir, trava o scroll do fundo, restaura o
  foco ao fechar, e o ESC fecha só o modal do topo (e respeita
  `[closeOnBackdrop]`/novo `[closeOnEsc]`).
- **`gems-modal` tamanho**: use apenas `sm | md | lg` (o `xl` nunca existiu).
- **IDs internos**: geração de IDs agora funciona em `http://` e SSR (antes
  `crypto.randomUUID` podia quebrar).
- **Loading global**: o overlay não some mais prematuramente após `forceHide()`.

---

## 7. Novidades opcionais (i18n / customização)

- `gems-table` aceita `[labels]` para traduzir a paginação:
  ```html
  <gems-table [labels]="{ perPage: 'Per page:', next: 'Next' }" ... />
  ```
  e `[rowId]="'id'"` para `trackBy` estável.
- `gems-side-menu` aceita `[logoutLabel]`, `[logoutTitle]`, `[openMenuLabel]`, `[appName]`.
- `GemsAlertService.errorFromApi(err, { title, fallback })` permite textos custom.
- `gems-file-upload` aceita mensagens custom (`invalidFormatMessage`,
  `uploadSuccessMessage`, `uploadErrorMessage`).
- `gems-input-text`, `gems-textarea`, `gems-select` agora emitem `(valueChange)`.
