Auth & Keycloak

Autenticação via Keycloak é opcional — a SDK não exige a dependência. Instale keycloak-angular apenas quando necessário.

Configure o KeycloakService com um APP_INITIALIZER antes de usar o guard ou a diretiva.

Guard funcional que protege rotas por lista de roles Keycloak.

Diretiva estrutural que exibe/oculta elementos com base nas roles do usuário autenticado.

Totalmente opcional

GemsBaseStore injeta KeycloakService com { optional: true }. Sem o Keycloak instalado, as requisições são feitas sem token — ideal para APIs públicas ou autenticação própria.

Instalação

Use npm install keycloak-angular keycloak-js --legacy-peer-deps. O gemsRoleGuard e *HasRole lançam erro em runtime se chamados sem o serviço presente.
