Loading Interceptor

gemsLoadingInterceptor ativa o spinner automaticamente em cada requisição HTTP. GemsLoadingService permite controle manual quando necessário.

Registre o interceptor funcional via withInterceptors e adicione <gems-loading> ao template raiz.

Contagem por referência

O serviço mantém um contador interno. O spinner só desaparece quando todas as requisições em andamento terminam. Chamadas manuais a show()/hide() seguem a mesma lógica.

forceHide()

Em casos de erro crítico onde o contador perde a sincronia, use loadingService.forceHide() para zerar o estado.

Signal-based

loadingService.isLoading() retorna um Signal<boolean>, compatível com qualquer template Angular sem AsyncPipe.
