Integração IA

Guia de acessibilidade para configurar modelos de Inteligência Artificial no consumo da SDK.

A GEMS SDK foi construída pensando na escalabilidade do uso de IAs generativas (como Cursor, Claude Code, GitHub Copilot). Possuímos três mecanismos principais para ensinar seu assistente a consumir nossos componentes nativamente.

1. Padrão llms.txt

Nossa documentação fornece um mapa estruturado em llms.txt. Quando quiser que a IA conheça nossa SDK do zero, forneça a URL no prompt:

https://Gabriel-mDias.github.io/Angular-GEMS-SDK/llms.txt

2. Guia de Consumo Embutido (NPM)

Ao instalar a SDK via npm, baixamos automaticamente o arquivo de contexto na pasta do pacote. Adicione esta instrução no prompt da sua IA ou em regras globais do projeto:

Leia as diretrizes de código contidas no pacote em: /node_modules/@gabriel-mdias/angular-gems-sdk/AI-CONSUMER-GUIDE.md

3. .cursorrules / AGENTS.md

Para projetos em editores IA-native, como o Cursor, cole o snippet abaixo no arquivo .cursorrules na raiz do seu projeto consumidor:

# Integração GEMS SDK
Este projeto consome a biblioteca de componentes GEMS SDK. 
Antes de criar formulários, tabelas ou interfaces, LEIA OBRIGATORIAMENTE o guia de uso nativo da biblioteca disponível no seu próprio node_modules:
Caminho: `node_modules/@gabriel-mdias/angular-gems-sdk/AI-CONSUMER-GUIDE.md`

Utilize as regras contidas lá como verdade absoluta para a construção de telas e lógica neste projeto.
